# Akorith 0.9.0

Akorith 0.9.0 adds Autonomous Research as a first-class local workflow beside General Chat,
Workspace, and Loop. Describe the outcome once, choose a logged-in CLI model, depth, and output
format, then let the job plan, collect evidence, verify claims, synthesize, and publish without
waiting for follow-up questions.

## Highlights

- Concurrent Research tabs with Quick, Research, Deep, and Continuous modes.
- Explicit Claude, Codex/ChatGPT, OpenCode, or optional Ollama model selection through existing
  local CLI logins—no provider API keys stored by Akorith.
- Durable SQLite jobs, cycles, checkpoints, leases, events, sources, claims, and artifact revisions;
  interrupted work recovers after restart and Continuous jobs run until paused.
- Bounded web acquisition with private-network and redirect protection, prompt-injection
  containment, source deduplication, claim-to-source citations, and a retained evidence ledger.
- Validated Markdown, A4 PDF, DOCX, and print-fitted XLSX exports with checksums.
- A persistent book-style Library with generated A4 covers, status filters, artifact open/reveal,
  source inspection, pause/resume, and archive controls.
- Responsive dark/light UI, keyboard-safe tabs, reduced-motion support, and provider/model state
  isolation across concurrent jobs.
- Safer macOS refresh: a complete replacement app is staged and validated before the installed
  Akorith bundle is moved.

## Validation

- Research contract, input, network-safety, persistence/recovery, and artifact suites.
- 32 deterministic library combinations: four depth modes × two provider classes × four output
  formats, plus invalid-boundary cases.
- Visual inspection of PDF, DOCX, and every XLSX worksheet print page.
- A real persisted OpenCode free-model Research run with sources, claims, citations, and a ready
  artifact. Claude-class deterministic fixtures passed; a real Claude smoke requires an active
  local `claude` login.
- Full Workspace, Loop, controller, startup hydration, executor/runtime, update-version, plugin,
  OpenCode-output, Goal-cycle, Windows-identity, TypeScript, and production-build regression.

## Notes

Desktop artifacts are currently unsigned. On macOS, right-click the app and choose **Open** on the
first launch when Gatekeeper warns. Existing chats, projects, Goals, profile settings, and Research
Library data remain in the local Akorith application-data directory during an update.
